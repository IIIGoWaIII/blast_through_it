Known Issues

  1. Font path resolution: Fonts in subdirectories not found when CSS is at root. Need to resolve paths relative to CSS file location, not EPUB root.
  2. WOFF/WOFF2 support: Only TTF fonts extracted. Need to handle other formats or let browser handle via @font-face.
  3. Tag selector specificity: CSS specificity not handled. Later rules override earlier ones regardless of specificity.
  4. Descendant selectors: Selectors like .calibre_5 p not parsed — only bare class or bare tag selectors.
  5. CSS shorthand font property: font: italic bold 16px serif not decomposed into individual properties.

Readest Approach (Reference)

Readest uses foliate-js for EPUB parsing. Key patterns:

Architecture

  • foliate-js: Pure JS EPUB/MOBI/FB2 parser — handles archive, CSS, rendering, annotations
  • zip.js: ZIP extraction (alternative to JSZip used by epubjs)
  • iframe-based rendering: Each chapter rendered in isolated iframe with full CSS support
  • CSS cascade: Natural browser CSS engine handles specificity, inheritance, @font-face

Why Our Approach Differs

We extract text for speed-reading (RSVP / visual pacer), not full rendering. We need:

  • Plain text with paragraph breaks
  • Formatting metadata (alignment, font, italic) per paragraph
  • Image tokens for inline images
  • No iframe overhead — text processed in React component tree

Improvement Plan

Priority 1: Fix Font Loading

  • Resolve @font-face URLs relative to CSS file location (not EPUB root)
  • Try multiple font formats: TTF → WOFF2 → WOFF → OTF
  • Fallback: if font not found, skip gracefully (browser default)

Priority 2: Improve CSS Selector Support

  • Handle descendant selectors: .parent .child { ... }
  • Handle element+class selectors: p.intro { ... }
  • Basic specificity: inline > class > tag

Priority 3: Additional Formats

  • MOBI/KF8 support (via KindleUnpack or similar)
  • FB2 support (fictionbook XML format)
  • CBZ support (comic book ZIP)

Priority 4: Advanced Features (from Readest)

  • Dictionary lookup (inline or popover)
  • TTS integration
  • Annotation/highlight system
  • Full-text search across chapters
  • Parallel read (split view)

Notes

  • epubjs book.load(href) returns an XML Document via DOMParser.parseFromString(xhtml, "application/xhtml+xml")
  • XHTML tag names are lowercase when parsed as XML — hence EPUB_BLOCK_TAGS includes both cases
  • book.archive.getText() strips leading / then looks up in JSZip
  • CSS paths in <link> are relative to the XHTML file, not the EPUB root
  • Font paths in CSS url() are relative to the CSS file location
